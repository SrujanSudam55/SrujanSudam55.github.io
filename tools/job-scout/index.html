Job Scout

Job Scout is a local, single-user job discovery system built to answer one question well:

“Which roles are actually worth my attention today, and why?”

View the repo on GitHub Back to tools

The problem it solves

Most job alerts optimise for coverage, not relevance. They surface volume, repeat noise, and hide why something matched.

  • Signal decay Job boards prioritise freshness and reach, not fit.
  • Duplicate fatigue The same roles resurface across days with no memory.
  • Opaque matching No explanation of why a role is considered relevant.

Design principles

  • Local and reliable Runs locally on a schedule. No accounts, no hosting.
  • Relevance first Ranks jobs using role-specific examples and a CV.
  • Explainable by default Every match includes a human-readable reason.
  • Memory built in Deduplicates across days and tracks job lifecycle.

How it works

  • Curated source registry A manually maintained Excel file defines which company career pages are scraped.
  • Modular scraping Adapters for common ATS platforms with browser fallback for JS-heavy pages.
  • Normalized job schema All jobs conform to a single schema before storage or ranking.
  • Persistent local memory SQLite tracks first seen, last seen, and active status.
  • Role-specific ranking Jobs are scored using example JDs and CV text, with deterministic outputs.

Outputs

  • Daily Top 20 A clean, ranked list of the most relevant roles with links and reasons.
  • Full ranked list All matched roles for inspection and comparison.
  • Artifacts, not state Outputs are disposable. The database is the memory.

Why this exists

Job Scout is not a product. It is a thinking instrument.

It reflects how I approach systems: define boundaries, enforce schema, build memory, and optimise for repeatable usefulness.
